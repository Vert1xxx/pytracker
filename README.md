**Описание проекта**

`PyTracker V3` — набор инструментов/шаблонов для сбора системной информации и построения вспомогательных скриптов. В репозитории есть builder, который генерирует Python-скрипт на основе шаблона. Этот проект не предназначен для несанкционированного доступа, перехвата учетных данных или массового сбора чужих данных.

**Требования**

- Python 3.8+
- Рекомендуется запускать в виртуальном окружении

Примерный список зависимостей:
```
requests
pycryptodome
pystyle
psutil
pyautogui
manuf
```

Установить зависимости:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
```

**Установка и запуск**

1. Клонируйте репозиторий или поместите файлы в рабочую директорию.
2. Активируйте виртуальное окружение и установите зависимости (см. выше).
3. Запустите builder:

```powershell
python PyTracker.py
```

Builder предложит ввести параметры (например, токены/ID для легальных интеграций) и создаст Python-файл в указанном `output`.

**Сборка в .exe**

Если вы хотите собрать сгенерированный `.py` в `.exe`, используйте `PyInstaller`. Делайте это только с собственными/тестовыми скриптами.

Установка PyInstaller:

```powershell
pip install pyinstaller
```

Сборка:

```powershell
pyinstaller --onefile --windowed --name MyTool dist_input.py
```

После успешного выполнения исполняемый файл окажется в папке `dist`.

**Безопасные альтернативы и рекомендации**

Если ваша цель — анализировать и извлекать собственные email/Telegram-имена/контакты для восстановления или аудита, рассмотрите следующие безопасные подходы:

- Локальный парсер (рекомендуется): сканирует только файлы и директории, к которым у вас есть доступ, и извлекает email-адреса и возможные Telegram-хэндлы по регулярным выражениям. Такой инструмент полезен для очистки логов, восстановления данных и аудита.

- Telegram API (Telethon): только для доступа к данным вашего аккаунта после явной аутентификации. Не используйте для доступа к чужим аккаунтам.

- Gmail API (OAuth2): безопасный и поддерживаемый способ доступа к своей почте; требует настройки проекта в Google Cloud и получения согласия владельца аккаунта.

**Пример безопасного сценария**

1. Вы работаете над восстановлением собственных данных: запускаете локальный парсер, указываете папку с бэкапами — получаете список email-адресов и возможных хэндлов.
2. Вы хотите проанализировать свой Telegram-аккаунт: настраиваете `Telethon` и проходите OAuth/авторизацию, затем анализируете только свои сообщения/контакты.
